External auth provider for login and session. Backend (app.auth) checks Supabase for health, protected routes, profile; frontend uses Supabase client and createAuthHeaders() for authenticated API calls. No user data stored in Django DB for auth.

Core library (enhanced)
core/
Base Node, ports, schema, Workflow/WorkflowBuilder; no Django.
Core workflow engine in src/neuroworkflow/core/: base Node, typed ports, schema definitions, and Workflow/WorkflowBuilder for execution order and dependency resolution. No Django dependency; imported by backend and by examples.
Modules
  • node.py – Node base class, ProcessStep, execute(), execution lifecycle
  • port.py – InputPort, OutputPort, type checking, connect()
  • schema.py – NodeDefinitionSchema, ParameterDefinition, PortDefinition, ResourceRequirements, PortType (JSON-serializable)
  • workflow.py – Workflow, WorkflowBuilder; add_node(), connect(); execution order and dependency resolution
nodes/
Concrete nodes by category; drive NEST, TVB, SONATA.
Concrete node implementations in src/neuroworkflow/nodes/: each subpackage defines nodes that inherit from Node, expose NODE_DEFINITION, implement process steps and optionally generate_python_script() for code/notebook export. Drive simulation backends (NEST, TVB, SONATA) as Python modules.
Categories
  • network/ – populations, connections
  • simulation/ – execution nodes (NEST, TVB, etc.)
  • stimulus/, analysis/, optimization/, io/
Integration
  • Backend syncs node definitions from UPLOAD_NODES_PATH, NEURO_PATH; box app stores PythonFile
  • CodeGenerationService uses these node classes + WorkflowBuilder to produce Python/notebook
utils/
Script export, job managers, ParameterMetadataService, database adapters.
Shared utilities in src/neuroworkflow/utils/: script and SnakeMake export, job managers (e.g. SLURM), ParameterMetadataService (with optional OpenAI), and database_adapters. Backend imports ParameterMetadataService and uses it in app.metadata views.
Modules
  • script_exporter – Python script, .ipynb; snakemake_generator – Snakefile, config.yaml
  • job_managers – JobManager base, SLURMJobManager
  • parameter_metadata_service.py – ParameterMetadataService; optional OPENAI_API_KEY, aggregates adapter results
  • database_adapters/ – AllenBrain, NeuroMorpho, PubMed, NeuroML-DB, Generic, LocalRAG
  • mcp/server – used by gui/mcp_server
Database adapters
Adapters query external DBs and Local RAG; return ParameterSuggestion.
src/neuroworkflow/utils/database_adapters/: each adapter implements DatabaseAdapter (query_parameter, get_source_name) and returns List[ParameterSuggestion]. ParameterMetadataService aggregates results from enabled adapters; Django app.metadata calls the service for parameters/suggest and custom-databases.
Adapters
  • AllenBrainAdapter, NeuroMorphoAdapter – HTTP; optional OpenAI for mapping
  • PubMedAdapter (API key recommended), NeuroMLDBAdapter
  • GenericDatabaseAdapter – user-defined HTTP/API; config via api/metadata/custom-databases
  • LocalRAGAdapter – HTTP to LOCAL_RAG_BASE_URL (port 8006)
cli/
CLI entry point: neuroworkflow run, info; no backend required.
Command-line entry point in src/neuroworkflow/cli/. Entry point: neuroworkflow.cli.commands:main (e.g. neuroworkflow run <file.py>, info). Runs workflow scripts that use WorkflowBuilder and node classes; no backend required.
  • Commands: run, info, etc.; install via pip install -e .

Primary persistence for the Django backend. All app data stored via ORM: FlowProject, FlowNode, FlowEdge (workflow); PythonFile (box); CustomDatabase (metadata custom DBs). Docker service db; config DB_NAME, DB_USER, DB_PASSWORD, DB_PORT.

Port: 5432 (internal)
Allen Brain Atlas
External DB: AllenBrainAdapter, HTTP; parameter metadata for suggestions.
External API: Allen Brain Atlas. AllenBrainAdapter (database_adapters/allen_brain.py) queries it over HTTP; returns parameter suggestions (e.g. cell types, regions). No API key required; optional OpenAI client for AI-powered parameter mapping. Used by ParameterMetadataService when enabled.
NeuroMorpho
External DB: NeuroMorphoAdapter, HTTP.
External API: NeuroMorpho.org (neuronal morphologies). NeuroMorphoAdapter (database_adapters/neuromorpho.py) queries over HTTP; returns ParameterSuggestion for morphology-related parameters. No API key required; optional OpenAI for mapping. Enabled by default in ParameterMetadataService.
PubMed
External DB: PubMedAdapter.
External API: PubMed/NCBI. PubMedAdapter (database_adapters/pubmed.py) queries for literature-based parameter suggestions. API key optional but recommended for higher rate limits. Returns ParameterSuggestion with source and confidence; used by ParameterMetadataService.
NeuroML-DB
External DB: NeuroMLDBAdapter.
External API: NeuroML-DB. NeuroMLDBAdapter (database_adapters/neuroml_db.py) queries for NeuroML model and parameter metadata. Returns ParameterSuggestion list; integrated into ParameterMetadataService. Used for model/parameter discovery in workflow parameter UI.
Custom DBs
User-defined DBs via GenericDatabaseAdapter; CRUD at api/metadata/custom-databases.
User-defined databases via GenericDatabaseAdapter (database_adapters/generic.py). Config stored in Django (CustomDatabase model); CRUD and test-connection at api/metadata/custom-databases/. Each custom DB can expose HTTP/API; adapter maps queries to ParameterSuggestion. Managed from Web UI or API.
Local RAG
LocalRAGAdapter calls this over HTTP (port 8006) for semantic parameter suggestions.
Local RAG backend service (rag/), exposed on port 8006. LocalRAGAdapter sends HTTP requests (e.g. semantic query over papers/chunks); responses normalized to ParameterSuggestion. Backend sets LOCAL_RAG_BASE_URL; optional Bearer/auth. Used for parameter suggestions when RAG is running.
Simulation backends (enhanced)
NEST Simulator
Simulation backend: spiking neural networks; loaded as Python module by nodes/.
Simulation backend for spiking neural networks; used by workflow nodes (nodes/) as a Python module. Generated scripts or notebooks import NEST; JupyterHub single-user image typically includes NEST. No direct HTTP; core library and backend only drive it via generated code.
TVB (Virtual Brain)
Simulation backend: whole-brain, mean-field models, brain connectivity.
Simulation backend for whole-brain, mean-field models and brain connectivity. Used by workflow nodes (nodes/) as a Python module; generated code imports TVB. Examples in examples/ and notebooks/; JupyterHub image may include TVB. Loosely coupled; no API—invoked via generated Python.
SONATA
Network format standard; interoperability.
Network format standard for interoperability (SONATA format). Used by nodes/ for reading/writing network specifications; supports large-scale and multi-population models. Examples in examples/; loaded as library by generated workflow code. Not a service—used as a standard/data format.
Custom
Extensible: custom simulation backends as Python modules.
Placeholder for custom simulation or execution backends. The architecture allows additional Python modules or services to be invoked by workflow nodes (e.g. custom generate_python_script() or process steps). No fixed API; extensible via new node classes in nodes/.

Workflow execution flow

1 User creates workflow (Web UI or Python API)
2 Workflow stored in DB (Django, FlowProject/FlowNode/FlowEdge)
3 User executes or generates code
4 WorkflowBuilder computes execution order (core library)
5 For each node: Validate → Execute → Generate script → Update ports
6 Scripts exported (Python / Notebook / SnakeMake)
7 Parameter suggestions: Frontend → Backend → ParameterMetadataService → DB adapters (Allen, RAG, etc.)
8 Results available for analysis

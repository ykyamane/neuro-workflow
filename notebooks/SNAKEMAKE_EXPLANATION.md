## 📊 What's Happening

### Current Node Architecture

The SONATA nodes in the system are **process-based nodes**, not **code-generation nodes**:

1. **Process-Based Nodes** (current):
   - Execute methods directly in Python
   - Use `process()` method with internal logic
   - Work great for interactive workflows
   - ❌ Don't have `python_code()` or `notebook_code()` methods

2. **Code-Generation Nodes** (future enhancement):
   - Have explicit `python_code()` method
   - Can export standalone scripts
   - Better for HPC batch jobs
   - ✅ Would show `has_python: True`

## 🎯 What Gets Exported

### SnakeMake Files Generated:

Even without Python code fragments, the SnakeMake generator creates:

#### **Snakefile** (1607 bytes):
- Workflow rules for each node
- Node dependencies (which nodes run first)
- Resource requirements (CPUs, memory, time)
- Input/output file specifications
- Placeholder execution commands

#### **config.yaml** (430 bytes):
- Workflow parameters
- Node configurations
- Resource specifications
- File paths
